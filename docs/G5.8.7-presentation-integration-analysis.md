### 1.2 当前架构 vs 目标架构

```
当前（分散调用）                         目标（统一分发）

GameScene                                GameScene
  ├─ play_sfx("melee")                     └─ emit_presentation_event(...)
  ├─ eng._attack_effects += ...
  ├─ self.presentation.trigger_shake(8)          │
  ├─ self.presentation.spawn_themed_damage(...)  ▼
  ├─ self.audio.play_boss_phase2_cue()    PresentationSystemDirector
  └─ play_recipe("skill_ice_nova", ...)     ├─ BuildTheme → resolve colors
                                             ├─ Recipe → VFX list
                                             ├─ Timeline → sequence events
                                             ├─ Camera → shake/zoom/dash
                                             ├─ Audio → SFX + BGM layer
                                             └─ Damage Number → themed floats
```

### 1.3 `_add_skill_effect()` 的 if/elif 链是最坏耦合点

```python
# game_scene.py L600-640 — 9 branches, theme override hardcoded
if name == "斩击":         play_sfx("slash")
elif name == "神罚":        play_sfx("bolt")
elif name == "自愈":        play_sfx("heal")
elif name == "冰爆":        play_sfx("ice_crack"); preset="ice"    ← hardcoded
elif name == "连锁闪电":     play_sfx("lightning"); preset="lightning"
elif name == "暗影突刺":     play_sfx("shadow_step"); preset="shadow"
elif name == "血怒":         play_sfx("blood_frenzy"); preset="bleed"
elif name == "召唤英灵":     play_sfx("summon"); preset="summon"
```

**问题**:
- 新增技能 = 新增 elif 分支 = 修改 GameScene
- `preset="ice"` 硬编码了主题 → 与 `BuildTheme.vfx_preset` 重复
- SFX 名称与 Recipe 名称分散在两个 if/elif 链里

## 3. 目标架构设计

### 3.1 核心抽象：PresentationEvent

```python
@dataclass
class PresentationEvent:
    kind: str          # "skill_cast" | "melee_hit" | "boss_phase2" | "boss_landing" | ...
    cx: int = 0
    cy: int = 0
    target_cx: int = 0
    target_cy: int = 0
    dmg: int = 0
    dmg_is_magic: bool = False
    direction = None
    skill_name: str = ""
    skill_level: int = 1
    theme_override: str = ""     # "" = use active theme
    recipe_override: str = ""    # "" = auto-resolve from skill_name
    sfx_override: str = ""       # "" = auto-resolve from recipe
```

### 3.2 统一入口：PresentationSystemDirector.dispatch()

```
GameScene._handle_player_attack()
  → self.presentation.dispatch(PresentationEvent(
        kind="melee_hit", cx=..., cy=..., dmg=42, ...))

PresentationSystemDirector.dispatch(ev):
  1. BuildTheme.resolve(ev)        → colors
  2. Recipe.resolve(ev)            → recipe_name, sfx_name
  3. VFX = play_recipe(recipe, ...) → [effects]
  4. eng._attack_effects += VFX
  5. Camera.on_event(ev)           → shake/zoom/dash
  6. Audio.on_event(ev)            → play_sfx + ducking
  7. Damage.on_event(ev)           → spawn_themed_damage
```

### 3.3 Recipe → SFX 自动映射

在 `vfx_recipes.json` 中为每个 recipe 增加 `"sfx"` 字段：

```json
"skill_ice_nova": {
  "sfx": "ice_crack",
  "steps": [...]
}
```

## 6. 风险和约束

| 风险 | 缓解 |
|:---|:---|
| The World 时停分支逻辑特殊 | 时停的 SFX 在 `_activate_time_stop` 内，保留为特例，不强制走 dispatch |
| Monster AI 写 `_attack_effects`（L667） | Monster AI 继续直写 `_attack_effects` 列表，因为 AI 是 Gameplay 层，它的 effects 是"请求"不是"展示决策" |
| Timeline 内置 recipe 捕获 scene 引用 | Step 1 创建时传入 context dict 而非 scene 对象 |
