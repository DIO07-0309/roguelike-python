# Game Design Review — Roguelike 地牢肉鸽

> 基于 `game-design` skill 的 6 个模块对项目进行系统分析。
> 分析日期：2026-07-08

---

## 1. Core Loop Design（核心循环）

### 30 秒测试

按 skill 的 ACTION → FEEDBACK → REWARD → REPEAT 框架分析：

```
ACTION  →  WASD 移动探索房间 + 空格攻击怪物
FEEDBACK→  怪物受击闪光 / 粒子爆散 / 屏幕震动 / SFX
REWARD  →  掉落装备（4 稀有度彩色光效）/ 经验值 / 技能习得
REPEAT  →  清空房间 → 楼梯激活 → 下一层 → 更强怪物
```

| 维度 | 评价 |
|------|------|
| **循环密度** | 🟢 高 — 每个房间 1-2 只怪，12 个房间，30 秒内必然触发战斗→掉落循环 |
| **反馈即时性** | 🟢 优秀 — 受击闪光、死亡粒子爆散、掉落物脉冲光效、BGM 切换 |
| **奖励节奏** | 🟡 合理 — 50% 掉落概率 + 升级必获技能，不至于太频繁 |

### 各层级的嵌套循环

```
微观循环（30s）    房间探索 → 发现怪物 → 攻击 → 掉落 → 拾取
中层循环（2-3min） 清空房间 ×12 → 激活楼梯 → 下楼
宏观循环（15-30min）第1层 → Boss(5层) → Boss(10层) → Boss(15层) → 通关
元循环（多局）     死亡 → 继承存档/选关 → 挑战更高层 → 永久死亡重开
```

## 3. Player Psychology（玩家心理）

### Bartle 四型分析

按 skill 定义的 Achiever / Explorer / Socializer / Killer 框架：

| 玩家类型 | 当前游戏的满足点 | 评分 |
|----------|-----------------|------|
| **Achiever**（成就者） | 15 层通关、收集传说装备、全技能习得 | 🟢 强 |
| **Explorer**（探索者） | BSP 随机地图（每局不同）、隐藏房间？ | 🟡 中 |
| **Socializer**（社交者） | 无多人/排行榜/分享功能 | 🔴 弱 |
| **Killer**（竞技者） | Boss 战（3 个 boss × 独特机制）、无 PvP | 🟡 中 |

**诊断**：这是一个单机游戏，Achiever 满足度最高，Socializer 天然缺失（对单机 Roguelike 来说合理）。

### 奖励机制对照

| 奖励类型 | 项目中的实现 | 效果 |
|----------|-------------|------|
| **Fixed**（固定） | 升级必定 +ATK/DEF/HP + 获新技能 | 🟢 里程碑奖励，给玩家确定性 |
| **Variable**（可变） | 掉落物品稀有度（60%/25%/12%/3%），15% 杀怪习技 | 🟢 赌博快感，驱动反复刷怪 |
| **Ratio**（比率） | 经验公式 Level² × 20，渐进难度 | 🟢 前期快后期慢，合理 |

## 4. Difficulty Balancing（难度平衡）

### Flow State 分析

skill 的核心公式：

```
太难 → 挫败 → 退出
太易 → 无聊 → 退出
刚好 → 心流 → 沉浸
```

**当前项目的平衡手段**：

| 策略 | 实现 | 评价 |
|------|------|------|
| **Dynamic**（动态调整） | ❌ 未使用 — 难度完全由楼层预定义 | 可改进 |
| **Selection**（玩家选择） | ✅ 选关系统 + 存档继承，玩家可跳过已通关层 | 部分缓解 |
| **Accessibility**（无障碍） | ✅ 新手教程（6 步引导）+ 键位提示栏常驻 | 良好 |

### 难度曲线数据

```
楼层    1  2  3  4  [5-Boss]  6  7  8  9  [10-Boss] 11 12 13 14 [15-Boss]
HP倍率 1.0 1.15 1.3 1.5  1.0   1.6 1.8 2.0 2.2   1.0    2.4 2.7 3.0 3.3   1.0
攻击倍 1.0 1.12 1.25 1.4 1.0   1.5 1.65 1.8 2.0  1.0    2.2 2.4 2.6 2.9  1.0
怪物数  5   5   6   6   1(B)  6   6   7   7   1(B)   7   7   8   8   1(B)
```

**潜在问题** ⚠️：Boss 关怪物数为 1 且不缩放（倍率 1.0），但 Boss 本身有独特技能+狂暴机制。普通关卡难度线性递增，没有"休息层"（rest beats）。

### 改进建议

1. **加入"休息层"**：每 4 层后插入一个商店/回血房间（无怪物），给玩家喘息空间，符合 skill 的 *"Rest beats between intensity"* 原则。
2. **动态掉落补偿**：玩家 HP 低于 30% 时微调掉落概率，防止死亡螺旋。

## 5. Progression Design（进度设计）

### 四种进展维度对照

| 进展类型 | 项目实现 |
|----------|---------|
| **Skill**（玩家技术提升） | 🟡 教程教基础操作，但无高级技巧引导（如 kiting、技能连招） |
| **Power**（角色强度提升） | ✅ 等级属性增长 + 装备稀有度 + 技能升级 Lv1→5 + Buff 系统 |
| **Content**（新内容解锁） | ✅ 15 层 × 新怪物/Boss，选关解锁机制 |
| **Story**（叙事推进） | 🟡 Boss 有背景故事（介绍画面），但无主线剧情贯穿 |

### Pacing 原则对照

| 原则 | 现状 | 评分 |
|------|------|------|
| **Early wins**（快速上钩） | ✅ 第 1 层 5 只史莱姆，HP 仅 15，两刀一个 | 好 |
| **Gradually increase challenge** | ✅ 线性递增 + Boss 跳跃难点 | 好 |
| **Rest beats**（强度间歇） | 🔴 无休息层，连续 4 层普通关无喘息 | 缺失 |
| **Meaningful choices** | 🟡 装备替换（W/A/P）+ 技能槽位有限（4 主动），但无天赋树/Build路线 | 中等 |

## 6. Anti-Patterns（反模式检测）

按 skill 的 4 组 Don't / Do 对照：

| ❌ 反模式 | ✅ 正确做法 | 项目状态 |
|----------|-----------|---------|
| **Design in isolation** | Playtest constantly | 🟢 有 README 操作指南，说明经过实机测试 |
| **Polish before fun** | Prototype first | 🟢 M1→M13 渐进开发，先搞定核心循环再加系统 |
| **Force one way to play** | Allow player expression | 🟡 只有一种通关路线（清怪→下楼），无流派选择 |
| **Punish excessively** | Reward progress | 🟢 死亡保留存档 + 选关继续，非完全重来 |

### 额外发现

| 问题 | 严重度 | 建议 |
|------|--------|------|
| **无 Build 多样性** | 中 | 4 个主动技能+被动全叠属性，缺少组合流派（如火法/战士/召唤） |
| **药水/装备无策略深度** | 低 | 药水只有回血，可加攻速/护盾/隐身等消耗品丰富战术 |
| **无风险/回报权衡** | 中 | 每层必清房才能下楼，可加可选挑战（如精英怪掉更好装备） |
| **Boss 之外无"大时刻"** | 低 | 10+ 个普通房间体验同质化，可加随机事件/宝箱房/陷阱 |

```
┌──────────────────────────────────────────────┐
│  总体评分：B+（坚实的中型 Roguelike 骨架）      │
│                                              │
│  ★★★★☆ 核心循环 — 简单但完整，反馈即时        │

│  ★★★★☆ 代码质量 — 组件化+状态机+工厂模式      │
│  ★★★☆☆ 叙事表现 — Boss 有背景但无主线串联      │
└──────────────────────────────────────────────┘
```

**下一步可按 `game-design` skill 的建议优先改进**：
1. 增加休息层（rest beats）
2. 引入 Build 流派选择（如职业/天赋）
3. 丰富房间类型（宝箱/商店/陷阱/精英）
