## 七、AI 状态机

```
     玩家距离 > 视野
   ┌────────────────┐
   │      IDLE      │  随机八方向巡逻
   │    巡逻移动     │  2秒换向 / 30%停顿
   └───────┬────────┘
           │ 距离 ≤ 视野
           ▼
   ┌───────────────┐
   │     CHASE     │  向玩家直线移动
   │     追击       │
   └───────┬───────┘
           │ 距离 ≤ 攻击范围
           ▼
   ┌───────────────┐
   │    ATTACK     │  冷却就绪 → 攻击
   │    站定攻击    │  伤害结算
   └───────────────┘

Boss 额外:
  ATTACK 状态 → 每帧尝试放一个 Boss 技能 (暗影冲击/地裂/召唤)
  血量 < 40%  → ENRAGE (移速×1.6, 技能冷却×0.7, 永久)
```

## 八、设计模式应用

| 模式 | 应用位置 | 说明 |
|------|---------|------|
| **组件模式** | Entity + CombatStats + MonsterAI | 实体通过组合组件获得能力，不依赖继承 |
| **状态机** | GameEngine (TITLE→PLAYING→DEATH...), AIState (IDLE→CHASE→ATTACK) | 全游戏状态由状态机驱动 |
| **工厂模式** | spawn_monster(), spawn_boss(), generate_random_item() | 统一创建入口，隔离构造逻辑 |
| **观察者(监听)** | effects 列表传入 AI.update() | 怪物/Boss 攻击写入特效，引擎统一渲染 |
| **策略模式** | MonsterAI vs BossAI | Boss 替换 AI 实例获得特殊行为 |
| **单例(懒加载)** | BGM / SFX Sound 缓存 | 音效编译一次，复用循环 |

## 九、伤害公式

```
最终伤害 = max(1, (攻方ATK - 守方DEF × 0.5) × (±20%浮动))

受击方DEF 按攻击类型选择:
  AttackType.PHYSICAL → physical_defense
  AttackType.MAGICAL  → magical_defense
  AttackType.TRUE     → 0 (无视防御)
```
